# Чек-лист релиза ShiftAlarm

Воспроизводимая процедура выпуска: релиз не должен зависеть от памяти разработчика. Проходить
сверху вниз, ничего не пропуская. Для будильника цена «тихо сломали релиз» — пропущенные звонки.

## 0. Предпосылки (один раз настроить)

```text
- [ ] Подпись релиза: keystore + signingConfig. Ключи и пароли НЕ в репозитории
      (локальный keystore.properties / переменные окружения / CI-секреты). Сейчас release-сборка
      без signingConfig — настроить до первой публикации.
- [ ] Решить про minify/R8 (сейчас isMinifyEnabled = false). Если включать — проверить, что
      Room/Compose/рефлексия не ломаются, и обновить proguard-rules.pro.
```

## 1. Подготовка

```text
- [ ] Рабочее дерево чистое, всё нужное закоммичено.
- [ ] Поднять версию в app/build.gradle.kts: versionCode +1 (монотонно), versionName по смыслу.
- [ ] Обновить CHANGELOG (что изменилось для пользователя за релиз).
```

## 2. Проверки качества

```text
- [ ] ./gradlew testDebugUnitTest            — юнит-тесты движка/данных зелёные.
- [ ] ./gradlew lintDebug                     — линт без критичных предупреждений.
- [ ] ./gradlew :app:connectedDebugAndroidTest -Pandroid.testInstrumentationRunnerArguments.class=
      ru.titeha.shiftalarm.data.AppDatabaseMigrationTest   — миграции БД на устройстве.
- [ ] CI по последнему коммиту — зелёный (GitHub Actions).
```
(Команды сборки/тестов и переменные окружения — см. корневой README.)

## 3. Ручная проверка на устройстве

```text
- [ ] Пройти docs/MANUAL_TEST_MATRIX.md на реальном телефоне (хотя бы критичные разделы:
      базовые будильники, смены/ночные, периоды, надёжность/системные события, сигнал).
```

## 4. Сборка релиза

```text
- [ ] ./gradlew assembleRelease  (или bundleRelease для AAB в Google Play).
- [ ] Проверить, что артефакт ПОДПИСАН нужным ключом (apksigner verify).
- [ ] Release-сборка не тянет локальные случайные файлы; собирается с чистого клона.
```

## 5. Приватность и данные

```text
- [ ] Сверить с docs/BACKUP_AND_RESTORE.md: что бэкапится/исключено — актуально.
- [ ] Privacy policy (если публикуем в Google Play): у приложения нет серверов, кроме фонового
      чтения производственного календаря; личные данные — только в локальном/Google-бэкапе.
```

## 6. Публикация

```text
- [ ] Git tag версии (например v0.1.0) на релизном коммите.
- [ ] Сохранить артефакты (APK/AAB, mapping.txt если minify) вне рабочего дерева.
- [ ] Залить в канал распространения (Play / прямой APK).
```

## Что НЕ должно попадать в репозиторий

```text
- keystore и пароли подписи;
- локальные keystore.properties / local.properties с секретами;
- сгенерированные артефакты сборки.
```
