# Архитектура ShiftAlarm — какой файл за что отвечает

Карта проекта для навигации и изучения. Пакет: `ru.titeha.shiftalarm`.
Слои снизу вверх: **данные → движок (чистая логика) → планирование → UI → точки входа**.

Аналогия для C#/.NET: «движок» (`schedule/`) — это как чистая доменная библиотека без
зависимостей от платформы (тестируется xUnit-ом без эмулятора); `data/` — слой EF-подобного
доступа (Room ≈ EF Core); `ui/` — Compose ≈ XAML/WPF, но декларативно в коде.

---

## Данные — `data/` (хранение, Room ≈ EF Core)

| Файл | За что отвечает |
|------|-----------------|
| `AlarmEntity.kt` | Запись будильника (таблица `alarms`): время, режим (`weekly`/`shift`), маска дней, пресет/`cycleSpec`, опорная дата, флаги (разовый, заморозка цикла, `honorHolidays`, `polarity`). Плюс константы режимов/полярностей. |
| `AlarmPeriod.kt` | Период без будильника (таблица `alarm_periods`): отпуск/больничный/отгул/свой счёт. Тип хранится строкой в `reason`. FK на будильник, каскадное удаление. |
| `AlarmOverride.kt` | Пер-дневная правка календаря (таблица `alarm_overrides`): подмена смены на день/диапазон. Метод `toDayOverride()` разворачивает в доменный тип. |
| `AlarmDao.kt` / `AlarmPeriodDao.kt` / `AlarmOverrideDao.kt` | DAO — SQL-запросы (Room генерирует реализацию). CRUD + потоки (`Flow`) для живого UI. |
| `AppDatabase.kt` | Конфигурация БД Room: список сущностей, **версия схемы**, все **миграции** (`MIGRATION_1_2`…`MIGRATION_4_5`). Схемы экспортируются в `app/schemas/*.json`. |
| `AlarmRepository.kt` | Тонкая обёртка над DAO — единая точка доступа к данным для UI и планировщика. |

## Движок — `schedule/` (ЧИСТАЯ логика, без Android; здесь живут юнит-тесты)

Это ядро. Ни один файл здесь не зависит от Android — поэтому всё покрыто быстрыми юнит-тестами
(`app/src/test/.../schedule/`).

| Файл | За что отвечает |
|------|-----------------|
| `ShiftSchedule.kt` | Модель графика смен + **`ShiftEngine`** — резолв «какая смена/во сколько звонок на дату». Типы: `ShiftType`, `ShiftCategory`, `ShiftPattern`, `TemporarySwap`, `OffPeriod`, `ShiftSchedule`. Приоритет: исключение > подмена > отпуск > ротация. `nextAlarm(...)` — ближайший звонок (опционально с производственным календарём). |
| `AlarmTimes.kt` | Мост «один будильник → ближайшее срабатывание». `next(...)` выбирает ветку по режиму, учитывает периоды, правки, полярность и праздники. `nextWeekly` — режим по дням недели. `shiftBase` — берёт цикл из `cycleSpec` или пресета. |
| `ShiftPresets.kt` | Встроенные графики: `2x2`, `sutki` (сутки/трое), `mdn` (утро/день/ночь 3×2). **Здесь заданы времена звонков и раскладка ночей (Вариант Б).** |
| `ShiftCycle.kt` | Свёртка/развёртка цикла в «прогоны» (`SlotRun`, `group`/`expand`) — для компактного редактора «×N». |
| `ShiftCycleCodec.kt` | Сериализация произвольного цикла `List<ShiftType>` ↔ строка (`cycleSpec`). Формат `cat|time|name`, обратная совместимость. |
| `ScheduleOverrides.kt` | Раскладка пользовательских правок в поля движка (один день → исключение, диапазон → подмена). `cancelNight` — «умная отмена ночи» (снять звонок предыдущего дня). |
| `PeriodKind.kt` | Тип периода (отпуск/больничный/отгул/свой счёт) ↔ строка `reason`. |
| `VacationSick.kt` | Правило ТК РФ: больничный внутри отпуска **продлевает** отпуск (режет + переносит хвост). |
| `ShiftCalendar.kt` | Классификатор дня для календаря: `kindOf(date) → DayKind` (цвет/тип). |
| `ProductionCalendar.kt` | Производственный календарь: статус дня `WORKING`/`NONWORKING` (Сб/Вс + праздники + переносы). |
| `ProductionCalendars.kt` | Данные календарей по стране/году. Сейчас встроенный **РФ-2026** (переносы черновые — сверить). |
| `HolidayAlarms.kt` | Полярность `WORK`/`REST` + расчёт «простого» будильника по календарю (буди по рабочим / по выходным). |
| `SetAlarmSpec.kt` | Чистый перевод extras системного `ACTION_SET_ALARM` в `AlarmEntity` (постановка будильника Ассистентом). |

## Планирование и звонок — `alarm/` (мост к системе Android)

| Файл | За что отвечает |
|------|-----------------|
| `AlarmScheduler.kt` | Ставит/отменяет будильники через системный `AlarmManager.setAlarmClock`. Подгружает периоды/правки из репозитория, зовёт `AlarmTimes.next`. |
| `AlarmReceiver.kt` | Приёмник срабатывания: запускает звонок и планирует следующий повтор. |
| `AlarmService.kt` | Foreground-сервис, играющий сигнал независимо от UI. Кнопка/смахивание «Стоп», подъём экрана. Не «липкий» (не воскресает сам). |
| `BootReceiver.kt` | После перезагрузки телефона перепланирует все включённые будильники. |

## Интерфейс — `ui/` (Jetpack Compose ≈ декларативный XAML)

| Файл | За что отвечает |
|------|-----------------|
| `AlarmEditorScreen.kt` | Экран редактора будильника. Режимы, `WeeklyEditor`, `ShiftEditor` (календарь + правки + диалог правки дня/диапазона), `HolidaySection` (учёт праздников). **Самый большой UI-файл.** |
| `ShiftCycleEditor.kt` | Редактор произвольного цикла смен (слоты: категория + тумблер звонка + время). `defaultAlarmFor(category)` — время по умолчанию. |
| `ShiftCalendarView.kt` | Наглядный календарь-месяц: покраска по типу дня, точка-звонок, листание, тап по дню (интерактив). |
| `VacationPeriods.kt` | Секция периодов (`VacationSection`) + диалог добавления (`AddPeriodDialog`) с выбором дат и типа. |

## Точки входа (корень пакета)

| Файл | За что отвечает |
|------|-----------------|
| `MainActivity.kt` | Главный экран: список будильников + переход в редактор. Загрузка/сохранение (периоды, правки), обработчик `ACTION_SHOW_ALARMS`. |
| `AlarmActivity.kt` | Полноэкранный экран «Подъём!» с кнопкой «Стоп» (показывается при срабатывании). |
| `SetAlarmActivity.kt` | Обработчик системного `ACTION_SET_ALARM` (Ассистент/сторонние приложения ставят будильник «через нас»). |

## Как это работает вместе (поток)

1. Пользователь редактирует будильник в **`AlarmEditorScreen`** → сохраняется через
   **`AlarmRepository`** в **Room** (`AppDatabase`).
2. **`AlarmScheduler`** берёт будильник + периоды + правки, зовёт **`AlarmTimes.next`**, а тот —
   **`ShiftEngine`** / `HolidayAlarms`, чтобы вычислить ближайший звонок, и ставит его в
   системный `AlarmManager`.
3. В нужный момент срабатывает **`AlarmReceiver`** → запускает **`AlarmService`** (звук) и
   **`AlarmActivity`** (экран «Стоп»), затем планирует следующий.
4. После перезагрузки **`BootReceiver`** восстанавливает все будильники.

## Где что искать

- **«Во сколько зазвонит»** → `schedule/AlarmTimes.kt` + `schedule/ShiftSchedule.kt` (ShiftEngine).
- **«Как рисуется календарь»** → `ui/ShiftCalendarView.kt` + `schedule/ShiftCalendar.kt`.
- **«Как добавить поле в будильник»** → `data/AlarmEntity.kt` + миграция в `data/AppDatabase.kt`.
- **«Ночные смены / времена»** → `schedule/ShiftPresets.kt` (раскладка) и `ScheduleOverrides.cancelNight`.
- **«Праздники»** → `schedule/ProductionCalendar*.kt`, `HolidayAlarms.kt`.

> Тесты движка — `app/src/test/.../schedule/` (быстрые, без эмулятора).
> Тесты миграций БД — `app/src/androidTest/.../data/AppDatabaseMigrationTest.kt` (нужен телефон).
> Команда сборки/тестов — см. `README.md` (важен `GRADLE_USER_HOME`).
