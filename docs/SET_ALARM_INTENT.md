# Обработка внешнего ACTION_SET_ALARM

## Назначение документа

Этот документ фиксирует правило обработки внешнего intent `ACTION_SET_ALARM`.

Этот intent может прийти от системного приложения, Ассистента или стороннего приложения. Поэтому его параметры нужно считать недоверенными и проверять до создания записи будильника.

## Точка входа

За обработку отвечает `SetAlarmActivity`.

Activity читает extras из intent и передаёт их в чистую функцию `SetAlarmSpec.toAlarm`.

`SetAlarmSpec` не зависит от Android API и покрывается обычными unit-тестами.

## Проверяемые параметры

Проверяются следующие значения:

```text
hour
minute
message
calendarDays
```

## Час

Час обязателен.

Если часа нет, будильник не создаётся. Приложение открывает основной экран, где пользователь может создать будильник вручную.

Допустимый диапазон:

```text
0..23
```

Если час меньше 0 или больше 23, будильник не создаётся.

## Минуты

Минуты должны быть в диапазоне:

```text
0..59
```

Если минуты меньше 0 или больше 59, будильник не создаётся.

Если minutes extra отсутствует во внешнем intent, Android-обвязка передаёт значение 0, что соответствует началу часа.

## Подпись будильника

Подпись будильника приводится к безопасному виду:

```text
пробелы в начале и конце убираются;
слишком длинная подпись обрезается до 120 символов.
```

Пустая подпись допустима.

## Дни повтора

Дни повтора приходят в формате java.util.Calendar:

```text
Calendar.SUNDAY = 1
Calendar.MONDAY = 2
...
Calendar.SATURDAY = 7
```

Известные значения переводятся в внутреннюю маску дней недели.

Неизвестные значения игнорируются.

Если после фильтрации не осталось ни одного дня, создаётся разовый будильник.

## Поведение при ошибке

Если параметры некорректны, SetAlarmSpec.toAlarm возвращает null.

В этом случае SetAlarmActivity не сохраняет будильник и открывает основной экран приложения.

## Что проверяется тестами

Unit-тесты проверяют:

```text
отсутствующий час;
некорректный час;
некорректные минуты;
разовый будильник без дней повтора;
перевод Calendar-дней в маску;
игнорирование неизвестных дней;
нормализацию подписи.
```

## Правило для будущих изменений

Новые параметры из внешнего intent нельзя напрямую записывать в AlarmEntity.

Сначала они должны пройти через чистый слой валидации, который можно проверить unit-тестами.
