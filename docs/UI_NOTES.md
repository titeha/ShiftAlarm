# Заметки по UI/UX

Живой документ решений по интерфейсу. UI дорабатывается пакетами замечаний; сюда пишем принятые
принципы и заметные изменения, чтобы направление не расплывалось.

## Направление

```text
- «картинки, не текст»: где можно — иконки вместо текстовых кнопок/подписей;
- меньше визуального шума на главном экране, ключевые действия под пальцем;
- разрушительные/необратимые действия — только с подтверждением.
```

## Тема и настройки

```text
- тема — Material3, единая точка входа AppTheme (ui/theme): светлая/тёмная по выбору
  (Система/Светлая/Тёмная), на Android 12+ по умолчанию динамические цвета Material You;
  резервная фирменная синяя палитра на старых версиях;
- AppTheme оборачивает весь UI (MainActivity, AlarmActivity) — дальнейший полиш идёт поверх,
  без перекраски;
- настройки — экран SettingsScreen (шестерёнка в шапке списка); пока тема, дальше добавятся
  страна праздников, правила отпуска и т.п. Хранение — SettingsStore (SharedPreferences);
- компонент AnalogTimePicker — правку под тему вносит Артемий (его пакет).
```

## Главный экран (список будильников)

```text
- добавление — круглая плавающая кнопка «+» (FAB, CircleShape) в правом нижнем углу, поверх списка;
  список имеет нижний отступ, чтобы последний будильник не прятался за кнопкой;
- удаление — иконка-корзина (без текста);
- удаление будильника — с подтверждением (сложную смену жалко стереть случайным касанием);
- выключение будильника — с подтверждением; включение — сразу;
- вверху — «Диагностика» и баннер готовности (только при проблемах, критичное красным,
  рекомендации мягче).
```

## Редактор будильника (единый экран, 4 секции)

```text
Не «два режима-продукта», а один редактор с секциями (заголовок + разделитель):
- «Основное»    — название + тумблер «Включён»;
- «Когда звонить» — три чипа способа: «Разово» / «По дням недели» / «По графику»,
  под ними контент выбранного способа. Переключение чипа не стирает введённое в других
  способах: черновики маски дней / цикла / «удалить после срабатывания» живут в одном
  draft (AlarmEntity держит все поля), в entity сохраняется только активный способ (effective());
- «Когда не звонить» — «Учитывать праздники» (+ полярность), а для графика ещё периоды
  (отпуск/больничный/…) и «Заморозить цикл на время отпуска». Для weekly периоды пока
  не показываем — движок их не умеет (см. non-goals ТЗ), место в секции подготовлено;
- «Проверка» — превью SchedulePreview + сворачиваемый блок «Календарь и правки» (по умолчанию
  свёрнут; только для графика). Превью считается ТОЛЬКО движком (AlarmTimes/ShiftEngine), поэтому
  совпадает с реальным звонком (периоды, правки, праздники учитываются одинаково):
  · строка «Следующий звонок: дата · время» для всех способов (боевой AlarmTimes.next);
  · «по дням недели» — ближайшие 3 срабатывания списком;
  · «по графику» — мини-лента 14 дней (цвет типа + колокольчик, если звонит) и строки ближайших
    звонков «Пн 13 · Утро · звонок 05:30»; для ночи — «звонок накануне, 21:00».
  Данные строк даёт ShiftEngine.nextAlarms → AlarmOccurrence (ringAt/servedDate/shift/eveningBefore),
  чтобы «накануне» приходило из движка, а не пересчитывалось в UI. Сборку расписания для превью и
  календаря делает общий AlarmTimes.shiftScheduleOf (без дублирования).

Маппинг способов на entity (схема БД не меняется):
- «Разово»        = MODE_WEEKLY, пустая маска дней (явный выбор вместо скрытой конвенции);
  показывает тумблер «Удалить после срабатывания» (по умолчанию вкл при выборе способа);
- «По дням недели» = MODE_WEEKLY, непустая маска;
- «По графику»    = MODE_SHIFT.

Время — компактно: крупная кнопка-время открывает пикер по тапу (общий TimeButton).
Быстрые пресеты дней: «Все» / «Будни» (Пн–Пт) / «Выходные» (Сб–Вс) / «Очистить».
```

### График: сегментная лента цикла + bottom sheet

```text
- цикл смен показан СЕГМЕНТНОЙ ЛЕНТОЙ (CycleStrip): горизонтальный ряд блоков, ширина ∝ числу
  дней (Modifier.weight(n)), цвет фона — палитра типов из календаря (реюз colorOf/onCellColor,
  без новой палитры). Внутри блока «У ×3» + колокольчик, если звонит. Выбранный блок — рамка-акцент;
- тап по блоку открывает BlockSheet (ModalBottomSheet): тип (сегменты Утро/День/Ночь/Выходной),
  «Дней подряд» (степпер 1..30), «Будильник» (тумблер + время через AnalogTimePicker), название —
  в сворачиваемом «Дополнительно» (по умолчанию авто-имя по типу). Действия «Сдвинуть влево/вправо»,
  «Дублировать», «Удалить» — внутри sheet, не постоянные кнопки на ленте;
- ФИКСАЦИЯ ПРАВОК БЛОКА — явная: правки в sheet копятся ЛОКАЛЬНО и применяются только по галке ✓
  (в шапке sheet справа); крест ✕ (слева) или смахивание вниз — откат. Так «сохранилось/не
  сохранилось» очевидно. Итог всё равно пишется кнопкой «Сохранить» внизу всего экрана
  (sheet-галка фиксирует блок в черновик цикла, экранный Save — записывает будильник);
- «+ блок» вставляет блок «умного» типа — отличного от соседних, чтобы он не слился при нормализации
  и был сразу виден — и сразу открывает его в sheet для правки. Крест ✕ откатывает правки к дефолтному
  «умному» блоку (он уже в цикле), галка ✓ — сохраняет, «Удалить» — убирает. Слово «Запись» → «Блок»;
- смена типа: на «Выходной» выключает будильник, на рабочий — включает (можно переопределить);
  для ночи со звонком — подпись «Звонок накануне вечером, в HH:MM», для выходного — «Выходной со звонком»;
- ЯКОРЬ ЦИКЛА — вопросом, а не датой: вместо инженерного поля «опорная дата» под лентой цикла
  стоит вопрос «Сегодня, ср 15 июля — какая смена?» + чипы типов, присутствующих в цикле. Если тип
  встречается один раз — якорь ставится сразу; если несколько — «Который из этих дней — сегодня?»
  показывает цикл квадратиками (цвет типа + номер дня), кандидаты активны с подписью «N-й», прочие
  приглушены (~40% alpha). После выбора — summary «Сегодня: Утро (2-й из 3) · Изменить». В
  «Дополнительно» — альтернатива для вахты «Начало цикла с даты» (DatePicker, можно будущую).
  Математика — чистый CycleAnchor (опорная дата = сегодня − индекс дня; инвариант «сегодня резолвится
  в выбранный день», покрыт тестами, включая границу года и ночной эдж). Существующие будильники со
  старым anchor показывают вычисленный summary;
- пресеты — точка входа «возьми шаблон и поправь»: под чипом пресета мини-полоска цветов цикла;
- НОРМАЛИЗАЦИЯ (ShiftCycle): соседние блоки склеиваются, если совпадают тип/будильник/время/имя.
  Авто-именованный блок при смене типа переименовывается и получает время по типу → склейка работает;
  блоки с разными пользовательскими именами не сливаются. Покрыто юнит-тестами.
```

### На будущее: интернационализация «будни/выходные»

```text
Пресеты «Будни/Выходные» сейчас жёстко Пн–Пт / Сб–Вс. Реальность сложнее:
- в разных странах рабочая неделя и выходные разные (напр. в Японии, по словам Артемия,
  6-дневка с началом в воскресенье и одним выходным — субботой; проверить);
- если подключён модуль праздников: «выходные» = выходные + праздники (по производственному
  календарю), «рабочие» = только официально рабочие. Без модуля — только номинальные дни недели.
Отложено: сделать пресеты страна-/календарь-зависимыми.
```

## Календарь смен

```text
- индикатор звонка в дне — иконка-колокольчик (Icons.Filled.Notifications из material-icons-core),
  раньше была точка. Тот же колокольчик используется в сегментной ленте цикла — единая иконография
  «звонит будильник». Пустой контур (Outlined) потребовал бы material-icons-extended (тяжёлая
  зависимость) — ТЗ запрещает новые зависимости, поэтому филлед-колокольчик;
- цвета типов дней — палитра colorOf/onCellColor (переиспользуется лентой цикла).
```

## Экран звонка

```text
- текст — название будильника (поле «Название»), иначе дефолт «Подъём!»;
- сигнал (мелодия) — системный будильник по умолчанию; выбор своей мелодии — на будущее;
- запасной сигнал — вибрация (если звук недоступен, событие пишется в диагностику).
```
