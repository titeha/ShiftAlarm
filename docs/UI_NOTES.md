# Заметки по UI/UX

Живой документ решений по интерфейсу. UI дорабатывается пакетами замечаний; сюда пишем принятые
принципы и заметные изменения, чтобы направление не расплывалось.

## Направление

```text
- «картинки, не текст»: где можно — иконки вместо текстовых кнопок/подписей;
- меньше визуального шума на главном экране, ключевые действия под пальцем;
- разрушительные/необратимые действия — только с подтверждением.
```

## Тема и настройки

```text
- тема — Material3, единая точка входа AppTheme (ui/theme): светлая/тёмная по выбору
  (Система/Светлая/Тёмная), на Android 12+ по умолчанию динамические цвета Material You;
  резервная фирменная синяя палитра на старых версиях;
- AppTheme оборачивает весь UI (MainActivity, AlarmActivity) — дальнейший полиш идёт поверх,
  без перекраски;
- настройки — экран SettingsScreen (шестерёнка в шапке списка); пока тема, дальше добавятся
  страна праздников, правила отпуска и т.п. Хранение — SettingsStore (SharedPreferences);
- компонент AnalogTimePicker — правку под тему вносит Артемий (его пакет).
```

## Главный экран (список будильников)

```text
- добавление — круглая плавающая кнопка «+» (FAB, CircleShape) в правом нижнем углу, поверх списка;
  список имеет нижний отступ, чтобы последний будильник не прятался за кнопкой;
- удаление — иконка-корзина (без текста);
- удаление будильника — с подтверждением (сложную смену жалко стереть случайным касанием);
- выключение будильника — с подтверждением; включение — сразу;
- вверху — «Диагностика» и баннер готовности (только при проблемах, критичное красным,
  рекомендации мягче).
```

## Редактор будильника (единый экран, 4 секции)

```text
Не «два режима-продукта», а один редактор с секциями (заголовок + разделитель):
- «Основное»    — название + тумблер «Включён»;
- «Когда звонить» — три чипа способа: «Разово» / «По дням недели» / «По графику»,
  под ними контент выбранного способа. Переключение чипа не стирает введённое в других
  способах: черновики маски дней / цикла / «удалить после срабатывания» живут в одном
  draft (AlarmEntity держит все поля), в entity сохраняется только активный способ (effective());
- «Когда не звонить» — «Учитывать праздники» (+ полярность), а для графика ещё периоды
  (отпуск/больничный/…) и «Заморозить цикл на время отпуска». Для weekly периоды пока
  не показываем — движок их не умеет (см. non-goals ТЗ), место в секции подготовлено;
- «Проверка» — строка «Следующий звонок: дата · время» (из боевого AlarmTimes.next, не дубль
  логики) и сворачиваемый блок «Календарь и правки» (по умолчанию свёрнут; только для графика).

Маппинг способов на entity (схема БД не меняется):
- «Разово»        = MODE_WEEKLY, пустая маска дней (явный выбор вместо скрытой конвенции);
  показывает тумблер «Удалить после срабатывания» (по умолчанию вкл при выборе способа);
- «По дням недели» = MODE_WEEKLY, непустая маска;
- «По графику»    = MODE_SHIFT.

Время — компактно: крупная кнопка-время открывает пикер по тапу (общий TimeButton).
Быстрые пресеты дней: «Все» / «Будни» (Пн–Пт) / «Выходные» (Сб–Вс) / «Очистить».
```

### На будущее: интернационализация «будни/выходные»

```text
Пресеты «Будни/Выходные» сейчас жёстко Пн–Пт / Сб–Вс. Реальность сложнее:
- в разных странах рабочая неделя и выходные разные (напр. в Японии, по словам Артемия,
  6-дневка с началом в воскресенье и одним выходным — субботой; проверить);
- если подключён модуль праздников: «выходные» = выходные + праздники (по производственному
  календарю), «рабочие» = только официально рабочие. Без модуля — только номинальные дни недели.
Отложено: сделать пресеты страна-/календарь-зависимыми.
```

## Экран звонка

```text
- текст — название будильника (поле «Название»), иначе дефолт «Подъём!»;
- сигнал (мелодия) — системный будильник по умолчанию; выбор своей мелодии — на будущее;
- запасной сигнал — вибрация (если звук недоступен, событие пишется в диагностику).
```

## На будущее (кандидаты)

```text
- иконизировать режимы (недели/смены), типы периодов, дни недели;
- своя мелодия/громкость/нарастание;
- тёмная тема, доводка палитры календаря.
```
