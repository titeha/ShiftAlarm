# Диагностика будильника (журнал событий)

## Зачем

Будильник — вещь, где важен ответ на вопрос «почему не зазвонил / почему зазвонил». Логи в
консоли недоступны обычному пользователю и стираются. Поэтому приложение ведёт постоянный журнал
ключевых событий, который видно прямо на экране диагностики.

## Что записывается

| Событие | Когда | Откуда |
|---------|-------|--------|
| Запланирован | будильник поставлен на конкретное время | `AlarmScheduler.reschedule` |
| Сработал | срабатывание прошло проверку актуальности и запустило сигнал | `AlarmReceiver` |
| Пропущен | срабатывание отклонено (устаревшее/выключен) | `AlarmReceiver` |
| Перепланирован | после reboot / обновления приложения / смены времени/часового пояса | `BootReceiver` |

## Хранение

```text
- постоянный кольцевой буфер в SharedPreferences (до 200 событий, старые вытесняются);
- формат/обрезка — в чистом AlarmEventLogCodec (юнит-тесты);
- битые строки при чтении молча пропускаются.
```

## Экран

Кнопка «Диагностика» в шапке списка будильников открывает экран:

```text
- статус разрешений (всё в порядке / чего не хватает — подробности в баннере на главном экране);
- журнал событий, самые свежие сверху (время · тип · детали);
- «Очистить».
```

## Слои

| Компонент | Слой | Роль |
|-----------|------|------|
| `data/AlarmEvent` (+ `AlarmEventLogCodec`) | чистая логика | Типы событий и сериализация/обрезка. Юнит-тесты. |
| `data/AlarmEventLog` | данные | Постоянный буфер в SharedPreferences (`record`/`recent`/`clear`). |
| `ui/DiagnosticsScreen` | UI | Просмотр журнала и статуса готовности. |

## Возможные улучшения

```text
- показывать на экране и «следующий звонок сейчас» по каждому будильнику (снимок состояния);
- отмечать в журнале причину пропуска подробнее;
- экспорт/поделиться журналом для поддержки.
```
