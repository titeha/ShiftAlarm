# Ночные смены и привязка будильников

## Назначение документа

Этот документ фиксирует правило работы ночных смен в движке расписаний ShiftAlarm.

Правило важно для пользовательских циклов смен, потому что ночная смена отображается в календаре одним днём, а будильник для неё должен сработать вечером предыдущего календарного дня.

## Основное правило

День, отмеченный как ночная смена, считается днём отработки ночной смены.

Будильник для такой ночной смены должен звонить накануне, в установленное пользователем время.

Пример:

```text
24 июня: выходной, звонок 21:00
25 июня: ночная смена
```

Это означает, что звонок 24 июня в 21:00 будит пользователя на ночную смену, которая в календаре отмечена 25 июня.

## Почему это важно

Для обычных дневных смен день смены и день звонка совпадают.

Для ночных смен это не так:

```text
Дневная смена:
  день D: смена + звонок утром этого же дня

Ночная смена:
  день D: звонок вечером
  день D+1: ночная смена в календаре
```

Поэтому движок должен различать:

```text
календарный день звонка
```

и

```text
день смены, которую этот звонок обслуживает
```

## Встроенный шаблон

Во встроенном шаблоне `Утро / День / Ночь` это правило уже было заложено вручную.

Звонки ночных смен были расположены в предыдущих слотах цикла. Поэтому встроенный шаблон отображал ночные смены корректно.

## Пользовательский цикл

В пользовательском цикле пользователь задаёт время будильника прямо у ночной смены:

```text
Ночь, 21:00
```

Для пользователя это естественная запись: “на эту ночную смену будить в 21:00”.

Но внутри движка `wakeTime` относится к календарному дню звонка. Поэтому перед расчётом расписания пользовательский цикл нормализуется.

## Нормализация пользовательских ночных смен

Если в пользовательском цикле слот имеет категорию `NIGHT` и содержит `wakeTime`, этот звонок переносится на предыдущий календарный слот.

Сама ночная смена остаётся на своём календарном дне, но её собственный `wakeTime` очищается.

Пример исходного пользовательского цикла:

```text
Выходной
Ночь 21:00
Ночь 21:00
Ночь 21:00
Выходной
```

После нормализации для движка:

```text
Выходной 21:00
Ночь 21:00
Ночь 21:00
Ночь
Выходной
```

Это означает:

```text
звонок перед первой ночной сменой стоит на предыдущем дне;
звонок перед второй ночной сменой стоит на дне первой ночной смены;
звонок перед третьей ночной сменой стоит на дне второй ночной смены;
последняя ночная смена в блоке не содержит собственного звонка.
```

## Отображение в календаре

После нормализации точки будильников в календаре должны отображаться на календарных днях звонка, а не обязательно на днях самих ночных смен.

Для блока ночных смен это может выглядеть так:

```text
День 1: выходной + точка будильника
День 2: ночь + точка будильника
День 3: ночь + точка будильника
День 4: ночь без точки
День 5: выходной без точки
```

Точка на дне 2 относится не к ночной смене дня 2, а к следующей ночной смене дня 3.

## Выходные, отгулы и отмена ночной смены

Если день ночной смены отмечен как выходной или попадает в период без будильника, должен отменяться звонок, который обслуживает именно эту ночную смену.

Для ночной смены дня `D` это звонок на день `D-1`.

При этом звонок на день `D` не должен отменяться автоматически, потому что он может обслуживать следующую ночную смену `D+1`.

Пример:

```text
24 июня: звонок 21:00 -> будит на ночь 25 июня
25 июня: ночная смена + звонок 21:00 -> будит на ночь 26 июня
26 июня: ночная смена
```

Если 25 июня отмечено как выходной:

```text
24 июня: звонок должен быть отменён
25 июня: звонок должен остаться, если 26 июня остаётся рабочей ночной сменой
26 июня: ночная смена остаётся рабочей
```

## Как это реализовано в движке

Это правило действует для ДВУХ способов пометить день нерабочим:

```text
- правка календаря «Выходной» на ночь-дне — через ScheduleOverrides.cancelNight
  (снимает звонок предыдущего дня, сохраняет исходящий);
- период без будильника (отпуск/больничный/отгул) на ночь-дне — через движок.
```

`ShiftEngine.nextAlarm` глушит звонок по дню ОБСЛУЖИВАЕМОЙ смены (`servedDateForAlarmOn`), а не по календарному дню звонка. Так и праздники (производственный календарь), и периоды без будильника, отмеченные на день ночи `D`, снимают звонок вечера `D-1`, который эту ночь обслуживает.

История: до 2026-07-10 периоды без будильника ошибочно проверялись по дню звонка (`D-1`), из-за чего отпуск, отмеченный на день ночи `D`, не глушил звонок и будильник звонил в отпуске. Исправлено: периоды глушат по обслуживаемому дню, как и календарь.

## Проверенное поведение

Исправление проверено на пользовательском цикле с ночными сменами.

После нормализации:

```text
ночные смены отображаются корректно;
точки будильников стоят на днях фактического срабатывания;
выходной для ночной смены корректно отменяет нужный звонок;
следующая ночная смена не теряет свой звонок, если она остаётся рабочей.
```

## Ограничения текущей модели

Текущая модель хранит только один `wakeTime` на календарный день.

Если в один календарный день потенциально должны сработать два разных будильника, текущая структура цикла не может выразить это без расширения модели.

На текущем этапе это ограничение принято как допустимое, потому что основной сценарий ночных смен укладывается в правило “одна точка звонка на календарный день”.

## Связанные тесты

Поведение пользовательских ночных смен должно быть покрыто unit-тестами:

```text
одиночная кастомная ночная смена звонит накануне;
блок кастомных ночных смен раскладывает звонки на предыдущие дни;
день ночной смены, отмеченный выходным, отменяет звонок накануне;
следующая ночная смена сохраняет свой звонок, если она остаётся рабочей.
```

Конкретные тесты (`schedule/ScheduleOverridesTest.kt`):

```text
cancelNight_silencesServingAlarm_keepsOutgoing       — правка «Выходной» на ночи;
cancelNight_adjacentNights_composeCorrectly          — отмена двух ночей подряд;
offPeriod_onNightDay_silencesServingAlarm            — отпуск на день ночи глушит звонок накануне.
```
