# Производственный календарь в сменном режиме

## Назначение документа

Этот документ фиксирует смысл производственного календаря для будильника
«По графику».

## Основное правило

В сменном режиме источником времени звонка всегда остаётся сменный цикл:

```text
утро — своё время;
день — своё время;
ночь — своё время накануне;
отдельный слот может быть без звонка.
```

Производственный календарь не создаёт новое фиксированное время и не заменяет
времена цикла.

## Что означает «Учитывать праздники»

Для сменного графика эта настройка означает:

```text
звонить по временам сменного цикла;
не звонить, если день обслуживаемой смены является
официальным нерабочим днём по производственному календарю.
```

Для ночной смены проверяется день самой смены, а не календарный день вечернего
звонка накануне.

## Почему в сменном режиме нет «По выходным»

Полярность REST была разработана как простой календарный будильник с одним
фиксированным временем:

```text
в официальный нерабочий день звонить в HH:MM;
в официальный рабочий день молчать.
```

Сменный график имеет несколько времён и собственную ротацию. Если применить
старую REST-ветку к смене, движок игнорировал цикл и использовал скрытые
`AlarmEntity.hour/minute`. В результате пользователь настраивал, например,
05:00 / 13:00 / 21:00, а фактический звонок мог быть в невидимые 07:00.

Поэтому «По выходным» не показывается для режима «По графику».

Если пользователю нужен отдельный сигнал в официальные выходные, он создаёт
отдельный простой будильник. Несколько независимых будильников в одни сутки
допустимы.

## Совместимость со старыми данными

Старая сменная запись может содержать:

```text
mode = shift
polarity = REST
```

Такая запись не должна использовать скрытое общее время.

При чтении и расчёте она безопасно трактуется как:

```text
mode = shift
polarity = WORK
```

После повторного сохранения запись нормализуется в БД.

Миграция схемы не требуется: поля и формат таблицы не меняются.

## Согласованность интерфейса

Для сменного режима одинаковое правило используют:

```text
боевой AlarmTimes.next;
строка «Следующий звонок»;
мини-превью;
календарь смен;
сохранённая запись.
```

То есть UI больше не может показать времена смен, а системный планировщик —
поставить скрытое фиксированное время.

## Проверка тестами

Unit-тесты проверяют:

```text
сменный режим не поддерживает REST;
weekly/простой режим сохраняет REST;
старая сменная запись нормализуется в WORK;
смена использует время цикла, а не hour/minute;
официальный праздник и выходные глушат старую REST-запись как WORK;
простой REST-будильник не меняет прежнее поведение.
```

## Проверка на устройстве

В сменном режиме:

```text
пункт «По выходным» не показывается;
описание говорит, что официальные нерабочие дни глушат звонки графика;
предпросмотр сохраняет времена смен;
после сохранения и повторного открытия поведение не меняется.
```

## Правило для будущих изменений

Если появится полноценный календарный способ расписания
«По официальным рабочим/выходным дням», его следует оформить отдельным
пользовательским способом, а не скрытой полярностью сменного цикла.
