# Настройка телефона под агрессивные прошивки (автозапуск)

## Проблема (P1-1) — подтверждена на устройстве

Многие прошивки по умолчанию **запрещают автозапуск** приложений и агрессивно чистят фон
(Xiaomi/MIUI/HyperOS, Huawei, Oppo/Vivo, отчасти Samsung). Если автозапуск запрещён, система
**не доставляет приложению boot-broadcast** — после перезагрузки будильники не перевыставляются
вообще, до ручного открытия приложения.

**Проверено вживую на Xiaomi (HyperOS), 2026-07-16:** с выключенным «Автозапуском» после
`adb reboot` наш `BootReceiver` не отрабатывал (`dumpsys alarm` = 0), будильник, стоявший до
ребута, залоченным **не звонил**. Автозапуск был выключен по умолчанию — включение его вернуло
перепланирование. Это самый частый реальный сценарий «проспал» на Xiaomi, и он **важнее Direct Boot**:
без автозапуска не работает ни слой A Direct Boot, ни обычный `BOOT_COMPLETED` после разблокировки.

Отдельно: `LOCKED_BOOT_COMPLETED` на этом MIUI directBoot-ресиверу фактически не даётся
(процесс стартует, re-arm не происходит) — см. `DIRECT_BOOT.md`. Поэтому «звонок точно в срок,
пока телефон ещё залочен», на MIUI не достигается перевыставлением; это отдельная задача.

## Решение

Состояние автозапуска система программно не сообщает, поэтому проводим пользователя за руку.

- **`alarm/VendorSetup.kt`** (чистый, покрыт тестом): по `Build.MANUFACTURER` возвращает `VendorGuide`
  — имя прошивки, пошаговую инструкцию и кандидатов `пакет/класс` экрана автозапуска. Для не-агрессивных
  прошивок (стоковый Android/Pixel) — null (раздел не показываем).
- **`ui/VendorSetupScreen.kt`**: экран «Настроить телефон» — шаги + best-effort кнопки:
  «Открыть настройки автозапуска» (пробует кандидатов, откат на детали приложения),
  «Энергосбережение» (`ACTION_IGNORE_BATTERY_OPTIMIZATION_SETTINGS`), ссылка dontkillmyapp.com.
- **Подача:** скрываемая карточка «Настройте телефон для надёжности» вверху списка (для агрессивных
  прошивок), плюс постоянный доступ через Настройки → «Надёжность» → «Настроить телефон». Флаг
  «скрыто» — в `SettingsStore`.

Кандидаты экранов автозапуска зависят от версии прошивки, поэтому пробуются по очереди с откатом.
На проверенном Xiaomi/HyperOS кнопка открывает штатный экран автозапуска корректно.

## Границы

Мы не можем ни включить автозапуск за пользователя, ни узнать его состояние — только объяснить и
довести до нужного экрана. Это единоразовая настройка со стороны пользователя.
