# Источник производственного календаря (праздники и переносы)

## Назначение

Производственный календарь (праздники, выходные, перенесённые дни) нужен для полярности будильника
«учитывать праздники»: буди по рабочим (глушить праздники) или по выходным (звонить в праздники).
Данные страно- и год-специфичны и меняются постановлениями, поэтому их нельзя намертво зашивать в код.

## Главный принцип: звонок не зависит от сети

Будильник обязан срабатывать оффлайн. Поэтому:

```text
движок читает календарь ТОЛЬКО из локальных данных (кэш → встроенный набор);
сеть используется лишь для ФОНОВОГО обновления кэша.
```

Даже если сети нет никогда, приложение работает на встроенном наборе.

## Слои

| Компонент | Слой | Роль |
|-----------|------|------|
| `schedule/ProductionCalendar` | движок | Статус дня (рабочий/нерабочий). Чистый. |
| `schedule/ProductionCalendars` | движок | Встроенный набор РФ + `fromIsDayOff(year, data)` (парсер) + хук `source` и `resolve(country, year)`. |
| `data/HolidayApi` | данные | Тонкий `GET https://isdayoff.ru/api/getdata?year=YYYY&cc=ru` → сырая строка (на IO, ошибки → null). |
| `data/HolidayCalendarRepository` | данные | Кэш сырого ответа в SharedPreferences; `cachedCalendar` (парсит кэш), `refresh` (валидирует и пишет), `install` (подключает кэш к движку). |
| `ShiftAlarmApp` | приложение | При старте: `install()` + фоновый `refresh("RU", год)`; при успехе — перепланирование будильников. |

## Формат источника (isDayOff.ru)

Ответ — строка из цифр, по одной на каждый день года по порядку (1 января → 31 декабря):

```text
0 — рабочий день
1 — нерабочий (выходной/праздник)
2 — сокращённый предпраздничный (рабочий)
4 — рабочий день (перенос)
```

`fromIsDayOff` переводит это в `ProductionCalendar`: нерабочие будни → `holidays`, рабочие Сб/Вс →
`workingWeekends`. Длина строки строго сверяется с числом дней в году (защита от мусора/кодов ошибок).

## Как движок получает календарь

```text
AlarmTimes.next → ProductionCalendars.resolve("RU", год)
resolve: source(country, year)   // локальный кэш, если установлен Android-слоем
      ?: of(country, year)        // встроенный набор конкретного года
      ?: bundled(country)         // базовый встроенный набор страны
```

`source` по умолчанию `null` — движок остаётся чисто тестируемым (падает на встроенные данные).

## Известные ограничения / доводка

```text
- обновление идёт при каждом старте приложения (без троттлинга «раз в сутки») — можно добавить;
- resolve берёт календарь по году «сейчас»; даты следующего года у альманаха текущего года
  трактуются лишь по Сб/Вс (без праздников) — заметно только на стыке декабрь/январь;
- пока одна страна (РФ); мультистрана — отдельный шаг (per-alarm/глобальный выбор страны);
- встроенный РФ-2026 — резервный черновик; актуальные данные приходят из источника.
```
