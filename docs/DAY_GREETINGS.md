# «Настроение дня»: праздник дня и фраза дня

Документ для Claude Code. Опциональная фича для поднятия настроения: после
выключения будильника — карточка «какой сегодня праздник» + пословица/фраза дня.
Не влияет на надёжность звонка ни при каких условиях — это жёсткое требование.

---

## 0. Концепция

Две независимые опции, обе управляются из настроек (раздел «Настроение дня»):

- **Праздник дня** — все праздники на сегодня: государственные, профессиональные,
  международные, забавные. Не путать с производственным календарём: этот модуль
  ничего не глушит и не будит, он чисто информационный.
- **Фраза дня** — пословица/поговорка или цитата из общественного достояния.

Поверхности показа:

1. **Главная — карточка после «Стоп»**: показывается на экране после закрытия
   сессии звонка со статусом `dismissed` (интеграция со `SNOOZE_AND_REPEAT.md`:
   после снуза — не показывать, после `missed` — не показывать). Тап по карточке →
   bottom sheet: описание праздника, остальные праздники дня, кнопка «Поделиться».
2. **Вторичная — уведомление**, выключено по умолчанию. Строго **отдельный канал**
   `day_greeting` (IMPORTANCE_DEFAULT), никогда — канал будильника. Показывается
   в момент первого `dismissed` за день; если будильников сегодня не было —
   не показывается вовсе (мы будильник, а не календарь).

Дефолты: карточка — вкл, уведомление — выкл, обе опции отключаемы независимо.

## 1. Данные: офлайн, из assets, без сети

Файлы: `assets/greetings/holidays_ru.json`, `assets/greetings/phrases_ru.json`.
Локаль — в имени файла, лукап по локали устройства с фолбэком на `ru`
(в этой версии наполняется только `ru`; структура готова к странам).

```json
// holidays_ru.json — элемент массива, ДВА способа задать дату:
{ "id": "miner-day", "name": "День шахтёра", "kind": "professional",
  "rule": { "month": 8, "dayOfWeek": "SUNDAY", "ordinal": -1 },
  "description": "1–3 своих предложения." }
{ "id": "cat-day-ru", "name": "День кошек в России", "kind": "fun",
  "date": "03-01", "description": "..." }
// kind: state | professional | international | fun
// ordinal: 1..4 или -1 (последний такой день месяца)

// phrases_ru.json:
{ "text": "Терпение и труд всё перетрут." }
{ "text": "…", "author": "Козьма Прутков" }
```

**Контент агент не создаёт**: в репозиторий кладутся только 8–10 примерных записей
для тестов. Запрещено генерировать праздники/фразы самостоятельно и запрещено
выкачивать их с сайтов — датасет собирает владелец отдельно (лицензии!). Задача
агента — схема, валидатор и код.

Правила выбора:

- праздников в день может быть несколько: карточка показывает первый по приоритету
  `state > professional > international > fun`, остальные — в bottom sheet
  («и ещё N праздников»);
- фраза дня **детерминирована датой**: стабильный хэш строки `YYYY-MM-DD` по
  модулю размера списка — у всех пользователей в один день одна и та же фраза
  (это фича: тема для разговора в автобусе общая).

## 2. Чистая логика

`DayGreetingResolver(dataset)` → `fun forDate(date: LocalDate): Greeting(
holidays: List<Holiday>, phrase: Phrase?)` — без Android-типов.

Валидатор датасета (юнит-тест, гоняется на реальных assets):

1. JSON парсится; `id` уникальны; `name`/`description` непустые.
2. У элемента ровно одно из `date`/`rule`; `date` валиден (включая «02-29» —
   разрешить, показывать только в високосные); `ordinal` ∈ {1..4, −1}.
3. Все `rule` разрешимы для годов 2026–2030 (нет «5-го воскресенья»).
4. Хэш фразы стабилен (зафиксировать 2–3 golden-значения).

## 3. Этапы (коммиты)

- **Этап 0 — разведка**: экран после «Стоп» (из SNOOZE-интеграции), структура
  настроек. Короткий план, без правок.
- **Этап 1 — модель + резолвер + валидатор** + примерные записи.
- **Этап 2 — карточка + bottom sheet + «Поделиться»** (plain-text:
  «Сегодня — День шахтёра. „Терпение и труд всё перетрут." — Будильник работяги»)
  + раздел настроек.
- **Этап 3 — уведомление** в отдельном канале.

## 4. Ограничения

- Не трогать: путь звонка, движок `schedule/`, Room, каналы будильника.
- Никаких сетевых запросов — только assets.
- Мьют канала `day_greeting` не должен влиять на звонок (тест ниже).
- Новых зависимостей не добавлять (JSON — тем же способом, что уже используется
  в проекте; проверить в разведке).
- Каждый этап — отдельный коммит; `testDebugUnitTest` и `assembleDebug` зелёные.
- Сборка (Windows): `JAVA_HOME="C:\Program Files\Android\Android Studio\jbr"
  GRADLE_USER_HOME="H:\gradlecache" ./gradlew.bat testDebugUnitTest assembleDebug --console=plain`.

Backlog (не делать, оставить комментарием): выбор категорий праздников; «мой
профессиональный праздник» с персональным поздравлением; локали других стран
(через `CountryProfile`).

## 5. Ручная проверка

1. День с плавающим праздником (правило `ordinal`) — карточка корректна.
2. День без праздников — только фраза; обе опции выключены — карточки нет.
3. После снуза карточки нет; после финального «Стоп» — есть; после `missed` — нет.
4. Замьютить канал «Праздник дня» → будильник звонит как ни в чём не бывало.
5. «Поделиться» — текст уходит в мессенджер в заявленном формате.

## 6. Запуск

> Прочитай `docs/DAY_GREETINGS.md`. Этап 0 — покажи план. Далее этапы 1–3 по
> одному коммиту. Контент не генерируй — только схема, валидатор, примеры и код.
