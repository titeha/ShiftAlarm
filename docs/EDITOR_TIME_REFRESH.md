# Актуализация ближайшего звонка в интерфейсе

## Назначение документа

Этот документ фиксирует правило обновления времени в списке будильников
и в предпросмотре редактора.

Расчёт ближайшего звонка зависит от текущей минуты. Если экран долго остаётся
открытым, результат не должен продолжать показывать уже прошедшее срабатывание.

## Проблема

Ранний интерфейс использовал вызовы:

```text
LocalDateTime.now()
```

непосредственно во время композиции или сохранял значение через `remember`.

Из-за этого ближайший звонок обновлялся только при случайной повторной композиции:
например, после изменения настройки, возвращения на экран или обновления данных Room.

Если оставить редактор или список открытым, строка могла устареть.

## Принятое правило

На уровне UI используется один минутный таймер:

```text
rememberCurrentMinute()
```

Он:

```text
сразу возвращает текущую минуту;
обнуляет секунды и наносекунды;
ждёт до ближайшей границы минуты;
перечитывает системные часы;
обновляет Compose State;
повторяет цикл, пока экран находится в композиции.
```

## Почему точность до минуты достаточна

Настройки ShiftAlarm задаются с точностью до минуты.

Перерисовывать экран каждую секунду нет необходимости:

```text
это не улучшает расчёт расписания;
создаёт лишние recomposition;
увеличивает фоновую работу UI.
```

Обновление на границе минуты обеспечивает актуальный результат без секундного таймера.

## Единая временная точка

Одна и та же `LocalDateTime` передаётся:

```text
во все карточки списка будильников;
в строку «Следующий звонок»;
в список ближайших срабатываний редактора;
в сменный предпросмотр.
```

Так карточки одного экрана не рассчитываются в немного разные моменты.

## Изменение системного времени

Если пользователь вручную переводит часы, таймер может дождаться уже рассчитанной
границы. Максимальное запаздывание интерфейса ограничено одной минутой.

Системное планирование будильников отдельно перепланируется по `TIME_SET`
и не зависит от этого UI-таймера.

## Проверка тестами

Unit-тесты проверяют:

```text
обнуление секунд и наносекунд;
полную минуту на точной границе;
расчёт задержки в середине минуты;
одну миллисекунду перед следующей минутой;
защиту от нулевой задержки.
```

## Правило для будущих изменений

Компонент, показывающий относительный или ближайший момент звонка, не должен
самостоятельно вызывать `LocalDateTime.now()`.

Текущее время передаётся сверху из общего минутного состояния.
