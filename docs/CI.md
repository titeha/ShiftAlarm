# CI проекта ShiftAlarm

## Назначение

Этот документ фиксирует минимальную автоматическую проверку проекта.

CI нужен, чтобы каждый push и pull request проверяли базовую работоспособность проекта без ручного запуска команд на локальной машине.

## Что проверяется

Workflow `.github/workflows/ci.yml` запускается на:

```text
push в master/main;
pull_request в master/main;
ручной запуск через workflow_dispatch.
```

Проверяются три команды:

```text
./gradlew testDebugUnitTest --no-daemon --console=plain
./gradlew assembleDebug --no-daemon --console=plain
./gradlew assembleDebugAndroidTest --no-daemon --console=plain
```

## Зачем нужны эти проверки

testDebugUnitTest проверяет чистую логику:

```text
движок расписаний;
нормализацию ночных смен;
валидацию внешнего создания будильника;
проверку актуальности срабатывания;
защитное чтение сохранённых данных;
прочие unit-тесты.
```

assembleDebug проверяет, что debug APK собирается полностью.

assembleDebugAndroidTest проверяет, что инструментальные тесты компилируются. Это не запускает тесты на устройстве, но ловит ошибки компиляции Room-тестов и Android-test-кода.

## Что CI пока не делает

CI пока не запускает тесты на эмуляторе.

CI пока не запускает Android lint.

CI пока не собирает release APK/AAB.

CI пока не подписывает релизную сборку.

Эти проверки будут добавляться отдельными маленькими шагами.

## Артефакты

Workflow сохраняет:

```text
отчёты unit-тестов;
debug APK.
```

Отчёты нужны для разбора падений.

Debug APK нужен для быстрой ручной установки после успешной сборки.

## Правило для будущих изменений

Если в проект добавляется новая критичная логика, она должна быть покрыта тестом, который проходит в CI.

Если изменение требует проверки на устройстве, это фиксируется отдельно в документации ручной проверки.
