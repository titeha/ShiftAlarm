# Ввод названия будильника

## Назначение документа

Этот документ фиксирует поведение экранной клавиатуры при вводе названия будильника.

## Принятое правило

Поле названия передаёт клавиатуре следующие подсказки:

```text
текст является предложением;
в начале предложения предлагается заглавная буква;
кнопка действия клавиатуры отображается как «Готово»;
по нажатию «Готово» поле теряет фокус и клавиатура закрывается.
```

Для Compose используются:

```kotlin
KeyboardCapitalization.Sentences
ImeAction.Done
```

## Почему текст не изменяется программно

Приложение не должно принудительно менять регистр уже введённых символов.

Пользователь может намеренно использовать:

```text
аббревиатуру;
название со строчной буквы;
смешанный регистр;
имя на другом языке;
цифры и специальные символы.
```

Поэтому капитализация является подсказкой экранной клавиатуре, а не преобразованием строки в `onValueChange`.

## Ограничение платформы

Фактическое поведение зависит от установленной клавиатуры.

Большинство распространённых клавиатур учитывает `KeyboardCapitalization.Sentences`, но сторонняя клавиатура может проигнорировать подсказку. Это не считается ошибкой данных или редактора.

## Проверка

Поведение проверяется на реальном устройстве:

```text
открыть создание или редактирование будильника;
поставить курсор в пустое поле названия;
убедиться, что клавиатура предлагает заглавную первую букву;
ввести несколько слов;
убедиться, что регистр введённого текста не переписывается приложением;
нажать «Готово»;
убедиться, что клавиатура закрылась.
```

## Правило для будущих изменений

Если аналогичная подсказка понадобится другим полям названий, следует использовать те же параметры клавиатуры, не добавляя автоматическое преобразование текста.
