# Монетизация ShiftAlarm — стратегия

Рабочий документ. Фиксирует модель, границы бесплатного, лестницу шаринга и рельсы
оплаты по магазинам. Цифры в §6 — гипотезы для проверки, не решения.

---

## 1. Принципы (не нарушаем ни при какой модели)

1. **Надёжность бесплатна навсегда.** Всё, что влияет на «прозвенит / не прозвенит»
   (точные будильники, готовность, вибрация, самотест), — вне пейволла и вне рекламы.
2. **Реклама никогда не стоит на пути к звонку**: запрещена на экране звонка, экране
   готовности, в онбординге и в уведомлениях.
3. **Платит коллектив, а не одиночка.** Подписка оправдана серверными расходами.
4. Один пользователь — весь функционал бесплатно: несколько будильников, свой цикл,
   периоды, праздники, календарь, подмены. Это строит рейтинг и вирусность.

---

## 2. Сегменты и тарифы

### «Работяга» — бесплатно, навсегда
Всё индивидуальное ядро. Плюс бесплатный обмен графиком (уровни L0–L1, см. §3) —
это вирусная петля, её нельзя закрывать пейволлом.

### «Бригада» — подписка
Живой общий график с сервера (уровень L2). **Платит мастер/бригадир за бригаду**,
участники — бесплатно. Это соответствует и принципу §1.3, и реальности: у мастера
есть мотив (перестал отвечать на «а я завтра работаю?») и часто — бюджет.

### Донаты — параллельно, с первого дня
Кнопка «Поддержать» в «О приложении». Не открывает функций (важно для политик сторов).
Каналы: RuStore IAP-тиры (99/199/499 ₽) в RuStore-сборке; внешние (Boosty/ЮMoney) —
только там, где стор это разрешает (см. §5).

### Реклама — опция с триггером, не старт
Включать только после порога **N ≥ 10–20 тыс. DAU** (до этого — копейки при
репутационных издержках). Сеть: Яндекс/VK. Разрешённые зоны: низ списка будильников,
низ календаря. Запрещённые: см. §1.2, плюс редактор оставить чистым — показов там
всё равно почти нет. Если реклама включена — появляется разовая покупка
«Убрать рекламу» (это и есть мягкий индивидуальный Pro).

## 3. Лестница шаринга (техническая основа модели)

### L0 — share-код цикла (бесплатно, ближайший релиз)
- Формат: deep link `shiftalarm://plan?v=1&d=<base64url>` + QR на экране.
- Внутри — версионированный конверт JSON:
  `{ "v": 1, "type": "cycle", "payload": <строка ShiftCycleCodec> }`.
- Передача — **системный share sheet**: мессенджеры, а рядом лежащим телефонам —
  Quick Share (покрывает Bluetooth/Wi-Fi без единой строки нашего радио-кода).
  Свой Bluetooth/NFC/Wi-Fi Direct не писать: Android Beam выпилен из системы,
  прямые API дороги в поддержке и разрешениях.
- **Импорт — только с предпросмотром**: показать ленту ближайших дней через движок,
  применить по явной кнопке. Никакого автоприменения из ссылки.

### L1 — полный план (бесплатно)
- `type: "plan"`: цикл + якорная дата + периоды (+ настройки праздников).
- Тот же конверт, та же ссылка/QR; помещается в QR при разумном размере (< ~2 КБ),
  иначе — только ссылкой/файлом `.shiftplan`.
- Сценарии: «скинь свой график жене», «староста раздал группе расписание».

### L2 — живой график (подписка «Бригада»)
- Сервер: мастер правит план → у бригады обновляется + пуш; роли (мастер/участник);
  журнал изменений; позже — подмены смен между людьми («поменяться сменами»).
- Пуши: FCM как база + RuStore Push для устройств без GMS.
- **Не проектировать сервер сейчас.** Сейчас только не мешать будущему — см. §4.

---

## 4. Что заложить в код уже сейчас (дёшево, спасает потом)

- Конверт формата с полем `v` с первого дня — иначе первый же несовместимый импорт
  превратится в поддержку зоопарка.
- У плана — стабильный `uuid` и `updatedAt`: L2 станет «тот же план, но источник —
  сервер», а не новая сущность.
- Поле `origin` у графика: `local | imported | team` (пока используется только первое).
- Импорт-превью через тот же движок, что и планирование (переиспользовать функцию
  превью из ТЗ рефакторинга).

## 5. Рельсы по сторам (product flavors)

| Канал | Что можно | Ограничения |
|---|---|---|
| **RuStore** | IAP-донаты, подписка «Бригада» через RuStore Pay (СБП, карты); внешние платежи — можно, RuStore их не комиссует | Комиссия платёжного SDK 15%. С 01.02.2026 платёжные SDK доступны только ИП/ООО — **завести ИП до запуска платных функций** |
| **Google Play** | Бесплатная сборка для охвата (установки в РФ работают) | Биллинг для RU-разработчиков приостановлен → GP-flavor **без встроенных платежей вообще**; донат-ссылки наружу не встраивать (риск бана за обход биллинга) |
| **Сайт проекта** | Оплата «Бригады» через ЮKassa/аналог + активация по аккаунту | Для L2 аккаунты нужны в любом случае; вариант обхода комиссий. Юрлицо/ИП обязательны |

Практически: один код, флаги `flavor` решают наличие кнопок «Поддержать»/«Подписка».

## 6. Цены-гипотезы (проверить на живых пользователях)

- Донаты: 99 / 199 / 499 ₽ разово.
- «Бригада»: **199 ₽/мес или 1 990 ₽/год за бригаду до 15 человек** (платит мастер).
  Альтернатива для проверки: 490 ₽/мес до 50 человек (ЧОП/цех).
- «Убрать рекламу» (если реклама включена): 199 ₽ разово.

---

## 7. Триггеры и метрики (когда что включать)

| Событие | Действие |
|---|---|
| Релиз L0/L1 | Считать: доля пользователей, нажавших «Поделиться»; число импортов на один шаринг (K-фактор) |
| ≥ 10–20 тыс. DAU | Решение по рекламе (A/B: календарь vs список) |
| Устойчивые запросы «обновляемый график» в отзывах / ≥ X импортов | Старт проектирования L2 + оформление ИП |
| Запуск L2 | Северная звезда: активные бригады; конверсия «импортировал → в бригаде» |

---

## 8. Риски

- **Реклама ↔ доверие**: будильник просит отключить оптимизацию батареи; баннеры рядом
  с этой просьбой снижают конверсию в разрешения. Потому — триггер по DAU и зоны из §1–§2.
- **GP-сборка**: любые платёжные намёки — риск удаления; держать стерильной.
- **L2 — это второй продукт** (бэкенд, аккаунты, поддержка). Не начинать до сигналов из §7.
- **Шаринг-ссылки**: план от чужого человека — вектор пранка; предпросмотр обязателен,
  плюс подпись источника в будущем L2.
