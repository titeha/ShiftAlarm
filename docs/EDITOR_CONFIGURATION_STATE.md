# Состояние редактора при пересоздании Activity

## Назначение документа

Этот документ фиксирует первый этап сохранения состояния редактора будильника.

Цель этапа:

```text
не терять подтверждённый черновик при повороте экрана
и других конфигурационных пересозданиях Activity.
```

## Проблема

Ранний редактор хранил:

```text
AlarmEntity;
выбранный способ расписания;
периоды без будильника;
ручные правки;
флаг диалога отбрасывания
```

через локальные `remember` внутри `AlarmEditorScreen`.

При пересоздании Activity вся композиция строилась заново, и экран мог:

```text
вернуться в список;
потерять изменённое время;
потерять пользовательский цикл;
потерять периоды и правки;
сбросить dirty-state.
```

## Принятое решение

Одна сессия редактора представлена объектом:

```text
AlarmEditorSession
```

Сессия хранится в `AlarmListViewModel`.

ViewModel переживает конфигурационные изменения Activity, поэтому та же сессия
с теми же Compose State повторно передаётся новому экземпляру экрана.

В сессии находятся:

```text
исходные снимки;
текущий AlarmEntity;
текущие periods;
текущие overrides;
выбранный EditMethod;
флаг диалога отбрасывания;
расчёт currentAlarm;
расчёт dirty-state.
```

## Открытие редактора

Для нового будильника ViewModel сразу создаёт новую сессию.

Для существующего будильника ViewModel:

```text
отменяет предыдущую незавершённую загрузку;
читает periods и overrides;
создаёт сессию;
публикует её экрану.
```

Загрузка выполняется в `viewModelScope`, поэтому поворот Activity не отменяет её.

## Закрытие редактора

Сессия удаляется только при:

```text
явной отмене без сохранения;
подтверждённом отбрасывании;
успешном сохранении;
открытии другой сессии.
```

Во время состояния `Saving` закрытие сессии блокируется.

## Что сохраняется при повороте

Сохраняются подтверждённые изменения основного черновика:

```text
название;
время;
режим;
дни недели;
пользовательский цикл;
опорная дата;
праздничные настройки;
periods;
overrides;
dirty-state;
открытый диалог отбрасывания.
```

## Что пока не сохраняется

Локальное временное состояние вложенных контролов может закрыться:

```text
открытый выбор времени;
открытый DatePicker;
открытый bottom sheet блока;
неподтверждённые изменения внутри bottom sheet;
текущий выбранный месяц календаря.
```

Подтверждённые значения основного черновика не теряются.

## Уничтожение процесса

ViewModel не переживает полное уничтожение процесса.

Если Android выгрузит процесс в фоне, текущая сессия редактора пока не
восстанавливается. Это будет отдельный следующий этап через:

```text
SavedStateHandle;
компактный сериализуемый snapshot;
проверку размера сохранённого состояния.
```

## Проверка тестами

Unit-тесты проверяют:

```text
исходное состояние;
изменение AlarmEntity;
изменение метода;
изменение periods и overrides;
защиту исходных снимков от внешней мутации;
нормализацию legacy REST;
сохранение флага диалога.
```

Фактическое переживание поворота проверяется на устройстве.

## Правило для будущих изменений

Новое поле, влияющее на результат сохранения, должно находиться в
`AlarmEditorSession` либо в её сохраняемом snapshot.

Критичный черновик нельзя хранить только в локальном `remember`.
